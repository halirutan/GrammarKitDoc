# Section 3.3: Lexer Integration — Code Evidence

## 1. Lexer Generation Workflow Overview

**Source: `README.md:39-41, 219-243`**

Step-by-step workflow:
1. Create grammar `*.bnf` file.
2. Tune grammar using Live Preview.
3. Generate parser/ElementTypes/PSI classes (Ctrl-Shift-G / Cmd-Shift-G).
4. **Generate lexer `*.flex` file** and then **run JFlex generator** (both via context menu).
5. Implement `ParserDefinition` and register in `plugin.xml`.

> IElementType constants generated by the parser generator have to be recognized and returned by the lexer. The JFlex-based lexer can be generated from a grammar that defines all the required tokens ("Generate JFlex Lexer" menu).

> "Run JFlex Generator" menu in a `*.flex` file calls JFlex to generate lexer java code.

## 2. Token Definition in Grammar

**Source: `README.md:222-238`**

Tokens are defined in the `tokens` attribute:
```bnf
{
  tokens=[
    ...
    comment='regexp://.*'
    number='regexp:\d+(\.\d*)?'
    id='regexp:\p{Alpha}\w*'
    string="regexp:('([^'\\]|\\.)*'|\"([^\"\\]|\\.)*\")"
    ...
  ]
}
```

Keywords are picked from usages while tokens like `string`, `identifier`, and `comment` are defined with regexp patterns.

**Source: `resources/messages/attributeDescriptions/tokens.html`**

```bnf
{
  tokens = [
    id="regexp:\\w+"    // regexp token
    string             // no value or pattern
    PLUS_OP="+"
    SWITCH="switch"
  ]
}
```

## 3. BnfGenerateLexerAction: Flex File Generation

**Source: `src/org/intellij/grammar/actions/BnfGenerateLexerAction.java:63-266`**

### Activation
- Only enabled when current file is `BnfFile`.
- Template path: `/templates/lexer.flex.template`.

### Generation Process (lines 83-126)
1. Computes flex file name: `_<GrammarName>Lexer.flex` (line 258-263).
2. Searches for existing flex file with `FilenameIndex`.
3. Opens a save dialog (default: same directory as `.bnf` file).
4. Generates lexer text via Velocity template engine.
5. Saves text to the selected virtual file.
6. Associates `.flex` extension with `PlainTextFileType` if unknown.
7. Opens the generated file in the editor.

### Lexer Name Convention (lines 262-264)
```java
protected static String getLexerName(BnfFile bnfFile) {
    return "_" + BnfGenerateParserUtilAction.getGrammarName(bnfFile) + "Lexer";
}
```
Format: `_<GrammarName>Lexer` (prefixed with underscore).

### Token Processing (lines 138-169)
Two categories of tokens:
1. **Simple tokens** — literal text values (e.g., `PLUS='+'`).
2. **Regexp tokens** — pattern-based (e.g., `id='regexp:\\w+'`).

Additionally, any unquoted token references used in rules that don't match rule names are collected as implicit simple tokens.

### Java Pattern to JFlex Conversion (lines 218-256)

`token2JFlex()` method handles conversion:
- Regexp tokens: converted via `javaPattern2JFlex()`.
- Simple tokens: escaped via `text2JFlex()`.

Key regex conversions in `javaPattern2JFlex`:
| Java Pattern | JFlex Equivalent |
|---|---|
| `\\d` | `[0-9]` |
| `\\D` | `[^0-9]` |
| `\\s` | `[ \\t\\n\\x0B\\f\\r]` |
| `\\S` | `[^ \\t\\n\\x0B\\f\\r]` |
| `\\w` | `[a-zA-Z_0-9]` |
| `\\W` | `[^a-zA-Z_0-9]` |
| `\\p{Space}` | `[ \\t\\n\\x0B\\f\\r]` |
| `\\p{Digit}` | `[:digit:]` |
| `\\p{Alpha}` | `[:letter:]` |
| `\\p{Lower}` | `[:lowercase:]` |
| `\\p{Upper}` | `[:uppercase:]` |
| `\\p{Alnum}` | `([:letter:]|[:digit:])` |
| `\\p{ASCII}` | `[\\x00-\\x7F]` |

Forward slashes in patterns are quoted with double-quotes.

### Velocity Template Context (lines 191-207)
```java
context.put("lexerClass", getLexerName(bnfFile));
context.put("packageName", ...);
context.put("tokenPrefix", getRootAttribute(bnfFile, KnownAttribute.ELEMENT_TYPE_PREFIX));
context.put("typesClass", getRootAttribute(bnfFile, KnownAttribute.ELEMENT_TYPE_HOLDER_CLASS));
context.put("simpleTokens", simpleTokens);
context.put("regexpTokens", regexpTokens);
context.put("StringUtil", StringUtil.class);
context.put("maxTokenLength", maxLen[0]);
```

## 4. Lexer Flex Template

**Source: `resources/templates/lexer.flex.template` (48 lines)**

```flex
package $packageName;

import com.intellij.lexer.FlexLexer;
import com.intellij.psi.tree.IElementType;

import static com.intellij.psi.TokenType.BAD_CHARACTER;
import static com.intellij.psi.TokenType.WHITE_SPACE;
import static $typesClass.*;

%%

%{
  public $lexerClass() {
    this((java.io.Reader)null);
  }
%}

%public
%class $lexerClass
%implements FlexLexer
%function advance
%type IElementType
%unicode

EOL=\R
WHITE_SPACE=\s+

// regexp token macro definitions
#foreach( $token in $regexpTokens.keySet() )
$token=$regexpTokens.get($token)
#end

%%
<YYINITIAL> {
  {WHITE_SPACE}    { return WHITE_SPACE; }

  // simple tokens (literal matching)
  #foreach( $token in $simpleTokens.keySet() )
  "$simpleTokens.get($token)"    { return ${tokenPrefix}$token; }
  #end

  // regexp tokens (pattern matching)
  #foreach( $token in $regexpTokens.keySet() )
  {$token}    { return ${tokenPrefix}$token; }
  #end
}

[^] { return BAD_CHARACTER; }
```

Key features:
- Implements `FlexLexer` interface.
- Uses `%unicode` for full Unicode support.
- Returns `IElementType` instances.
- Imports from the generated `elementTypeHolderClass`.
- `BAD_CHARACTER` for unrecognized input.
- `WHITE_SPACE` for whitespace.
- Token constants are prefixed with `elementTypePrefix`.

## 5. BnfRunJFlexAction: JFlex Compilation

**Source: `src/org/intellij/grammar/actions/BnfRunJFlexAction.java:72-330`**

### Activation (lines 88-99)
- Enabled for `.flex` files or files with `.flex` extension.
- Also accepts `JFlexFileType.INSTANCE`.

### JFlex Execution (lines 128-186)
1. Parses `%class` directive to find lexer class name.
2. Parses `package` statement to find package name.
3. Resolves target directory for output.
4. Creates `SimpleJavaParameters` with:
   - JFlex jar path
   - `-Xmx512m` JVM parameter
   - `-skel` parameter pointing to `idea-flex.skeleton`
   - `-d` parameter for output directory
   - The flex file as input
5. Executes as an OS process with `OSProcessHandler`.
6. Shows output in Messages tool window console.

### JFlex Download (lines 254-311)
- JFlex URL: `https://cache-redirector.jetbrains.com/intellij-dependencies/org/jetbrains/intellij/deps/jflex/jflex/1.9.2/jflex-1.9.2.jar`
- Skeleton file: `idea-flex.skeleton` (extracted from the JFlex jar)
- Creates a global library named `"JFlex & idea-flex.skeleton"`.
- First checks existing global libraries for JFlex jars.
- Downloads if not found, creates/updates the global library.

### Console Output (lines 188-252)
- Uses IntelliJ Messages tool window.
- Content is keyed by batch ID for grouping multiple flex compilations.
- Reuses stopped process consoles when available.

## 9. Token Mapping: Grammar to Lexer

**Source: `README.md:219-243`**

The token mapping process:
1. Parser generator creates `IElementType` constants in `elementTypeHolderClass`.
2. The JFlex lexer must return these same constants.
3. The generated `.flex` template imports from the types class: `import static $typesClass.*;`
4. Each token rule returns `${tokenPrefix}$TOKEN_NAME`.

Example from the flex template:
```flex
"+"    { return PLUS_OP; }
{ID}   { return ID; }
```

## 10. Live Preview vs. JFlex Lexer Differences

**Source: `README.md:240-241`**

> While Live Preview mode supports full Java RegExp syntax and JFlex supports only a subset. Grammar-Kit tries to perform some obvious conversions.

## 12. Implicit Tokens

**Source: `README.md:176-185`**

Token types:
1. **Explicit tokens**: Declared in `tokens` attribute.
2. **Implicit keyword tokens**: Unquoted, name equals value.
3. **Implicit text-matched tokens**: Quoted, matched by text (slow — matched by text, not by IElementType).

Text-matched tokens can span more than one real token returned by the lexer.
